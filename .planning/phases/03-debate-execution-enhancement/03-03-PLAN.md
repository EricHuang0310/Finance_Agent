2. Implement `extract_trade_patterns(journal: list[dict], min_closed: int = 5) -> list[tuple[str, str]]`:
   - Filter to entries where `status == "closed"`
   - If fewer than `min_closed` closed trades, return empty list (cold start gate per research pitfall 3)
   - Group closed trades by `(sector, cio_stance, outcome)` tuple:
     * sector from `entry.get("entry_thesis", {}).get("sector", "unknown")`
     * cio_stance from `entry.get("entry_thesis", {}).get("cio_stance", "neutral")`
     * outcome from `entry.get("outcome", "unknown")`
   - For each group with >= 2 trades, compute:
     * avg_r = mean of r_multiple values (skip None)
     * avg_holding = mean of holding_days
     * avg_score = mean of entry_thesis.composite_score
   - Build situation-lesson tuple:
     * situation: `f"Sector={sector} CIO_stance={stance} trades={len(trades)} outcome_pattern={outcome}"`
     * lesson: `f"{outcome.upper()} pattern in {sector} during {stance} stance: {len(trades)} trades, avg R-multiple={avg_r:.2f}, avg holding={avg_holding:.1f} days, avg entry score={avg_score:.2f}. Recent symbols: {', '.join(t['symbol'] for t in trades[-3:])}"`
   - Also group by `(sector, order_type_used, outcome)` to learn fill quality patterns (EXEC-03 cross-reference):
     * Only if `order_type_used` field exists in entries (added by Plan 02)
     * situation: `f"Sector={sector} Order_type={order_type} outcome={outcome}"`
     * lesson: summary of fill quality by order type
   - Return list of (situation, lesson) tuples
   - Use `from collections import defaultdict` for grouping

3. Implement `load_and_extract_patterns(storage_dir: str = "memory_store", min_closed: int = 5) -> int`:
   - Loads journal from `logs/trade_journal.json` using `filelock.FileLock` (same pattern as trade_journal.py)
   - Calls `extract_trade_patterns(journal, min_closed)`
   - If patterns found, creates/loads `SituationMemory(name="trade_patterns", storage_dir=storage_dir)`
   - Clears existing patterns (they are re-derived each time from full journal) and adds new batch via `add_batch()`
   - Returns count of patterns extracted
   - Wrap in try/except, print warning on failure, return 0

4. Implement `get_pattern_memory(storage_dir: str = "memory_store") -> SituationMemory`:
   - Returns a SituationMemory instance for "trade_patterns" bank
   - Used by debate context prep to search for relevant patterns
